前言

版本控制：对软件开发过程中各种程序代码、配置文件及说明文档等文件变更的管理

集中式版本控制系统（CVS、SVN等）

SVN是集中式版本控制系统，版本库是集中放在中央服务器的，而工作的时候，用的都是自己的电脑，首先要从中央服务器得到最新的版本，然后工作，工作完后，需要把自己做的版本推送到中央服务器，而且集中式版本控制系统是必须联网才能工作。SVN采用增量式管理的方式保存每一个版本。

分布式版本控制系统（Git、BitKeeper等）

Git是分布式版本控制系统，它就没有中央服务器的，每个人的电脑就是一个完整的版本库，工作的时候不需要联网。Git采用文件系统快照的方式保存每一个版本。

分支

  • 分支的好处：
    同时多个分支可以并行开发，互相不耽误，互相不影响，提高开发效率。
    如果有一个分支功能开发失败，直接删除这个分支即可，不会对其他分支产生影响。

协同开发合作时解决冲突

​ 对于如下合作情景：
​ 1.开发人员A将创建的文件推送到远程库
​ 2.开发人员B将该文件从远程库克隆到本地
​ 3.开发人员B修改文件内容，并推送到远程库
​ 4.开发人员A修改文件内容，并推送到远程库

如果程序员A对该文件同一位置进行修复，将会产生冲突。
在冲突的情况下，应先拉取下来，然后修改冲突内容，然后再推送到远程服务器。

跨团队合作

1.得到远程库地址

2.进入合作者账号，打开链接，进行fork操作

3.克隆该链接中的库到本地，修改文件内容，然后推送到远程库

4.进行pull request操作

5.发起者通过审核进行合并操作（可在Files changed中查看提交的内容）

SSH免密操作

  • 在GitHub账户中添加你的公钥。（打开Git目录的git-bash.exe）

  • 在Github的settings中找到SSH and GPG keys设置选项，点击New SSH key，将你的id.rsa.pub里面的信息复制进去（Title任意）

  • 对SSH远程地址起别名

  • 通过ssh方式的地址推送到远程库

ps：使用ssh方式的好处：不要每次都进行身份验证，但只能针对一个账号
